Number Guesser Pt.2

Learn Basic Python Concepts by Building a Program that Guesses a Random Number of Your Choosing

In the previous post, Number Guesser Pt. 1, we built a program that allowed the user to guess a number randomly chosen by the computer. In this post, we will build a program that guesses a number in a pre-set range chosen by the user. This will teach us about functions, methods, f strings, and basic algorithm design.

To start, let’s decide a range that we can choose numbers from. For sake of ease and cliche, I’m choosing my range to be 1-100. Going much smaller than that allows the computer to guess the number too fast, but going much larger will take the computer too long to guess the number. Now that we have a range, let’s talk about algorithm design.

I see a few options. The first and most obvious is to have the computer start at 1 and guess in increments of 1 all the way up to 100. There’s a few reasons why this is a horrible algorithm to implement. First, it takes way too long. If the user chooses 97, for example, it will take 97 guesses to get the right number. Second, this algorithm, if implemented on a large scale, is very computationally intensive. We should always concern ourselves with developing programs that get to the solution in the least amount of steps possible. The third and probably most important reason for this project is that the algorithm is just not very impressive. I think we can design a systematic approach that just looks more engaging and fun.

While a few iterations of the previous algorithm exist, and some are better, the obvious choice for me was to use a method I’ll call ‘sandwiching.’ in this approach, we define an upper and lower bound. At first these bounds will assume the uppermost and lowermost value of range. From there we add those numbers and divide them by 2, if we get a decimal, we round down. If the user’s number is higher than the computer’s guess, the lower bound inherits the value of the computer’s guess. If the user’s number is lower than the computer’s guess, the upper bound inherits the value of the computer’s guess. This process is repeated until the computer ‘sandwiches’ the correct number. This method results in any number being guessed in just a few steps. I’ve provided a visual of what this process may look like below.

Fantastic. Now we have a range and an algorithm to guess the user’s number, all that’s left now is to code it up. Let’s start with a skeleton of what this code will look like. I believe we should do this code through two functions, a main, where the code will be called from, and a computer_guess() function, where the computer will decide the number. Once the computer_guess() function is completed, we know the computer is correct, so let’s notify the user.

All that we have to do now is code the computer_guess() function. Let’s start by defining 3 variables. One, and upper bound equal to 100. Two, a lower bound equal to 1. Finally, let’s define a variable called guessed which will notify us when the computer has successfully guessed the user’s number. Let’s set guessed equal to ‘False’ as the computer has not guessed the user’s number yet, this variable will be used in while loop a little later. Here are my variable declarations:

    lower = 1
    upper = 100
    guessed = False

Next let’s design the while loop. While guessed is false, we will have the computer continue to guess the user’s number. So to find the guessed number let’s create a variable called guess and set it to the integer value of the sum of the upper and lower bounds. That provides us with three options: the computer’s guess is higher, lower, or equal to the user’s number. Let’s ask the user to specify if the computer’s guess is ‘higher’, ‘lower’, or ‘equal’ to their number with the input() method. Next within each of the conditional statements, let’s adjust the bounds as previously explained.

Here’s the code for the computer_guess() function:

def computer_guess():
    lower = 1
    upper = 100
    guessed = False
    while not guessed:
        guess = int((lower+upper)/2)
        print(f'the computer guessed {guess}')
        relation = input('is this higher, lower, or equal to your number: ')
        if relation == 'higher':
            upper = guess
        elif relation == 'lower':
            lower = guess
        elif relation == 'equal':
            guessed = True

And that’s it! We’ve just coded a program that can guess a user’s number within the range 1 to 100. Here’s an image of my gameplay, my number was 76.
